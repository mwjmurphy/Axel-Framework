xml_to_bean - element

index

The root element for all xml to bean mapping.

attributes

child elements

ref- populator
minOccurs - 0
ref- bean
minOccurs - 1
maxOccurs - unbounded



bean - element

index

A bean defines the properties of a class included in the xml mapping.

attributes

element - attribute

type - xsd:string
use - required

The name of the xml element for this bean.

A creator may also be used to name the xml element for this bean.

clas - attribute

type - xsd:string
use - required

The name of the java class that this bean represents.

child elements

ref- creator
minOccurs - 0
ref- property
minOccurs - 0
ref- text
minOccurs - 0
maxOccurs - 1



creator - element

index

A creator is used when more than one xml element is managed by a bean. This is in addition to the bean element attribute.

attributes

element - attribute

type - xsd:string
use - required

The name of the xml element that this bean will manage.

child elements



property - element

index

A property is used to select a property from a bean class. The name of the property matches the bean name for the property.

Code Example:
private intanInteger







attributes

name - attribute

type - xsd:string
use - required

The name of the xml element that this bean will manage.

populator - attribute

type - xsd:string
use - optional

A String representation of a Java class that is used to populate the bean property from xml.

Used when a xml needs to be formatted or handled as a list, timestamp, date etc before population of the bean property.

populator_ref - attribute

type - xsd:string
use - optional

A reference to a populator.

Used when a xml to property needs to be formatted or handled as a list, timestamp, date etc.

child elements

ref- alias
minOccurs - 0



alias - element

index

An alias is used when an attribute name does not match the property name.

Each property may have multiple aliases.

attributes

name - attribute

type - xsd:string
use - required

The attribute name that is stored into the bean property.

child elements



populator - element

index

A populator is used to set the derived element attribute in the bean. Used when the object to be set in the bean is stored into a list, a date, a timestamp etc.

A set of populators are delivered with the product, but additional ones may be created and referenced from the configuration once they are set in the java classpath.

attributes

id - attribute

type - xsd:string
use - required

The reference identifier for this populater. Referenced from the property populator_ref attribute.

clas - attribute

type - xsd:string
use - required

The name of the Java Class that performs the population.

child elements

ref- keyvalue
minOccurs - 0



keyvalue - element

index

A keyvalue is used to store parameters for populators.

Example. when setting a timestamp or a date the format of the date may be set using key="time_format" value="yyyy-MM-dd HH:mm:ss.SSS"

attributes

key - attribute

type - xsd:string
use - required

The key to identifying this keyvalue

value - attribute

type - xsd:string
use - required

The value for the key.

child elements



text - element

index

A text element is used to store the element text into a bean property.

attributes

name - attribute

type - xsd:string
use - required

The name of the bean property where the element text will be stored.

child elements
