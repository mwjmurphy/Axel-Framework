code - element

index

The code action will invoke Java code from the web page.

If there is a response from the java call it will replace the code element in the page. If there is no response then the code element is removed from the page.

The response may also be stored in the exceContext using the key attribute.

attributes

use - required

<axel:code call="my.class.method"/>

Example with parameters
<axel:code call="org.xmlactions.common.date.DateUtils.getDate">
   <axel:param value="2010-10-02"/>

</axel:code>

insert - element

index

Insert files into a page.

The insert action is used to insert files into a page as it is being built on the server. The benefit of this is that you can separate out presentation objects into simpler pages and then combine them into the final presentation page before the page is rendered to the browser.

Notes

Inserted pages may contain their own actions. These actions will be executed before insertion of the page.

Inserted pages may also contain other inserted pages.

Object orientation of pages allows them to be shared by others.

attributes

type - xsd:string
use - required

The name of the file to insert.

The name of the file may also include the path location relative to the web context root.

path - attribute

type - xsd:string
use - optional

Additional path, prepended to the page.

By default it is set to the web context root.

namespace - attribute

type - xsd:string
use - optional
default - axel

Namespace used with pager action commands.

This allows you to declare the namespace to use for the inserted page. Usefull when you don't want to provide the html declaration.

remove_html - attribute

type - true_false_options
use - optional
default - true

If this is set true (which is the default setting) then the outermost html element is removed. This may be necessary when using strict conformant html.

insert_into - element

index

Insert the current processing page into another page.

This is used when you have an outer wrapper style page. The outer page could contain the head, body etc. The outer page must have a replacement marker key where the current page is inserted into. The marker must follow the standard ${insert_here} syntax. The "insert_here" can be any valid name. have a key placed where the this page should be inserted into.

Notes

One of the particular benifits of this action is the way that the page is called from the browser. Calling the inner page is very specific and can be easily traced from the browser.

Another benifit is the ability to have a consistant wrapper page that could contain title, menus, breadcrumbs etc. And having a number of pages inject themselves into the wrapper page. Each of the pages would only to be concerned about themselves, what they need to display and not be concerened about what they are in.

By default it is set to the web context root.

date_formatter - element

index

Convert a date from one format to another format.

This action takes a date in one format such as "2014-12-30" and converts it to anther format such as "30-DEC-2014". The formatter will handle both date and time individually or combined such as "2014-12-30 10:30" to "30-DEC-2014 10:30".

Some of the common date and time patterns in ‘SimpleDateFormat’ for Java String to Date conversion are:

  • Letter 'y' represents Year.
  • Letter 'M' represents Month in Year.
  • Letter 'D' represents Day in Month and ‘E’ is Day Name in Week.
  • Letter 'a' represents AM/PM marker and ‘h’ is Hour in AM/PM.
  • Letter 'H' represents Hour in a Day, ‘m’ represents Minute in Hour and‘s’ is Second in Minute.

Some Helpful Examples
"dd/MM/yyyy" = "12/05/2014"
"dd-MM-yyyy HH:mm:ss" = "12-05-2014 22:37:50"
"dd-MMM-yyyy" = "12-May-2014"
"MM dd, yyyy" = "05 12, 2014"
"E, MMM dd yyyy" = "Mon, May 12 2014"
"E, MMM dd yyyy HH:mm:ss" = "Mon, May 12 2014 23:37:50"
"dd-MMMM-yy"= "14-September-14"

type - xsd:string
use - required

This is the original date that we want to convert to another format.

origin_format - attribute

type - xsd:string
use - required

This is the format of the original date in java syntax. i.e. "dd/MM/yyyy" = "12/05/2014"

destination_format - attribute

type - xsd:string
use - required

This is the format that we want to convert the date to in java syntax. i.e. "dd-MMM-yyyy" = "12-May-2014"

range - element

index

Creates a Range bean that provides an incremental/decremental index or a random index.

It uses a from-to as the incremental/decremental range or the random range. If from < to the number will increment. If from > to the number will decrement.

THe range will start at "from" and increment/decrement until it reaches the "to" value. Then it restarts at the "from" continuing the loop. If from=5 and to=12 it will return 5,6,7,8,9,10,11,12 in a continuous loop. If from=12 and to=5 it will return 12,11,10,9,8,7,6,5 in a continuous loop.

For a random value it randomly picks a number between "from" and "to"

Example:

To apply the next range value simply add ${inc_1_10} to the page. This will be replaced with the next incremental number in the range 1-10. Each time the ${inc_1_10} is found it will be replaced with the next number.

attributes

name - attribute

type - xsd:string
use - required

This is the name of the range as it will be stored in the ExecContext. When referencing the range simply use ${range_name} to get the next value from the range.

from - attribute

type - xsd:string
use - required

This is the "from" value for the range.

to - attribute

type - xsd:string
use - required

This is the "to" value for the range.

random - attribute

type - xsd:boolean
use - optional
default - false

Set to true to generate a random number

Set to false to generate an incremental number

The file name of a presentation form to draw into.

A presentation_form will contain replacement markers that will be populated from the execContext.

For a list - each row from the query will be drawn into this form.
Use the ${...} replacement markers to insert the fields from the query into the form.

As an example to insert the value for field table.fieldName into the form use ${table.fieldName}.

When using JSON use a / as a seperator in place of a .

header_name - attribute

index

This is the presentation name for the header if displayed in a list.

expression - attribute

index

An expression to be evaluated using JavaScript.

If this expression evaluates to true then any enclosing actions are executed.

The expression may contain parameter references using the replacement markers. e.g. ${session:key}.

The characters &lt;, &gt; and &amp; must be used in place of their replacement characters <, > and &.

If a value for a replacement marker is not found it will be replace with a null.

uri - attribute

index

This is the uri (page) that gets called for this link. All input and option fields will be included in the page request. This attribute may be set to "" if the submit attribute is set "true", this will cause the containing action class such as add to hide the add from after a successfull submit.

display_as - attribute

index

Displays a link as a 'link' or as a 'button'. The selected theme for a link = INPUT_LINK and for a button = INPUT_BUTTON.
