The AXEL Framework is designed to run in a Java Web Server such as Tomcat or Application Server such as WebLogic. When it has been installed and configured it will service selective file types that are requested from a browser.

These are the file types that are usually serviced by AXEL

  • *.html - your html files
  • *.ajax - your ajax calls
  • *.bin - your binary files such as pdf, images etc

The installation also includes the spring framework which is used to construct an Execution Context, property settings, language settings, theme settings and any other configuration items you need for your application.

How to implement a Pre Processor Page


A pre processor page can be used to setup configuration items, validate users and other such functions required before any page processing begins. The pre processor page will be applied to all page types that have been defined in the servlet-mapping configurations. Define the name of the pre processing page by setting the file page name in the Pager Servlet configuration as shown below. 

    <init-param>
        <param-name>pager.pre.page</param-name>
        <param-value>pre.page.html</param-value>
    </init-param>
In the above example definition the pre.page.html is the file name of the pre processor page.
If during the page pre-processing you decide not to process any further you can stop the process by
  1. Set a property "pre.page.stop" to "stop" using <axel:put key="pre.page.stop">stop</axel:put>
  2. The content of the page will be displayed as the response.
  3. No other pages will be processed.

Spring Configuration Settings


The spring configuration is setup in the web.xml from the file classpath:/config/spring/spring-pager-web-startup.xml. This is used to setup the Application Execution Context commonly known as execContext. All beans stored here are available to the AXEL Actions.
